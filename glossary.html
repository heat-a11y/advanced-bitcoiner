Bitcoin Glossary

From Address to Zero-Conf — every term you need to know as an advanced Bitcoiner.

A
Address Basic
A Bitcoin address is a string of alphanumeric characters that identifies a destination for a Bitcoin payment. Modern addresses use Bech32 format (starting with bc1) for SegWit transactions, offering lower fees and better error detection.
Airdrop Token
Distribution of free tokens or coins to existing Bitcoin holders, often used as a marketing strategy by new blockchain projects to bootstrap their network. Recipients are typically selected based on a snapshot of on-chain balances.
ASIC Mining
Application-Specific Integrated Circuit — a microchip designed exclusively for Bitcoin mining. ASICs perform SHA-256 hashing at extremely high efficiency, making them the only viable hardware for profitable mining today.
ATH Trading
All-Time High — the highest price Bitcoin has ever reached in a given currency pair (e.g., BTC/USD). Each cycle produces a new ATH, typically occurring 12-18 months after the halving event.
B
Base58 Technical
An encoding scheme used in Bitcoin addresses and private keys (legacy format). It's similar to Base64 but excludes visually similar characters (0/O/I/l) and removes '+' and '/' to prevent accidental corruption when shared.
Bech32 Technical
A SegWit address format starting with 'bc1' that uses a more efficient encoding (Base32) and includes a BCH code for error detection. Bech32 addresses result in lower transaction fees and are the recommended format for modern Bitcoin wallets.
Bitcoin Core
The first decentralized digital currency, invented by Satoshi Nakamoto in 2008. Bitcoin enables peer-to-peer value transfer without intermediaries, secured by a proof-of-work consensus mechanism with a fixed supply of 21 million coins.
Block Core
A container of Bitcoin transactions grouped together and added to the blockchain approximately every 10 minutes. Each block references the previous block's hash, forming an immutable chain of transactions.
Blockchain Core
A distributed, immutable ledger consisting of blocks linked by cryptographic hashes. The Bitcoin blockchain records every transaction ever made and is maintained by a global network of nodes running the Bitcoin protocol.
Block Height Core
The number of blocks preceding a given block in the blockchain. Genesis block is height 0. Block height is used to determine confirmation depth and to schedule protocol changes (e.g., halvings occur every 210,000 blocks).
Block Reward Mining
The amount of newly created Bitcoin awarded to a miner for successfully mining a block. Currently 3.125 BTC per block (halved from 6.25 in April 2024). The reward halves every 210,000 blocks (~4 years).
BTC Core
The ticker symbol for Bitcoin on exchanges and trading platforms. 1 BTC equals 100,000,000 satoshis. BTC is used as a base trading pair on most cryptocurrency exchanges.
C
Change Address Technical
An address used to receive the "change" from a Bitcoin transaction when the inputs exceed the output amount. For example, if you have a 1 BTC UTXO but want to send 0.3 BTC, the remaining 0.7 BTC minus fees goes to a change address controlled by the sender.
Coin Control Privacy
The ability to manually select which UTXOs (unspent transaction outputs) to use as inputs in a transaction. Coin control allows users to manage privacy, optimize fees, and avoid consolidating high-value UTXOs that might reveal their full balance.
Cold Storage Security
A method of storing Bitcoin private keys completely offline, typically on hardware wallets, paper wallets, or air-gapped computers. Cold storage is the most secure way to hold significant amounts of Bitcoin long-term.
Confirmation Core
When a transaction is included in a block and subsequently built upon by additional blocks. Each additional block on top of the transaction's block counts as an additional confirmation. Most exchanges require 3-6 confirmations for deposits.
Consensus Core
The mechanism by which Bitcoin nodes agree on the state of the blockchain. Bitcoin uses Nakamoto Consensus, combining Proof-of-Work mining with the longest chain rule to achieve distributed agreement without a central authority.
D
DCA Strategy
Dollar-Cost Averaging — an investment strategy where a fixed dollar amount of Bitcoin is purchased on a regular schedule regardless of price. This reduces the impact of volatility and eliminates the need to time the market. The most recommended strategy for long-term Bitcoin accumulation.
Difficulty Mining
A measure of how hard it is to find a new Bitcoin block. The network automatically adjusts difficulty every 2,016 blocks (~2 weeks) to maintain the target 10-minute block interval. Higher difficulty means more hashing power is securing the network.
Dust Technical
Very small amounts of Bitcoin (typically under 546 sats) that cost more to spend than they are worth due to the minimum relay fee. Accumulating dust UTXOs can lead to bloated wallets and should be avoided.
E
Elliptic Curve Cryptography Technical
The mathematical foundation of Bitcoin's key generation. Bitcoin uses the secp256k1 elliptic curve to create public-private key pairs. The security relies on the computational infeasibility of the Elliptic Curve Discrete Logarithm Problem.
Exchange Trading
A platform where users can buy, sell, and trade Bitcoin for fiat currency or other cryptocurrencies. Major exchanges include Binance, Coinbase, Kraken, and Bitfinex. Non-custodial DEXs like Bisq offer peer-to-peer trading without KYC.
F
Fees Core
Transaction fees paid to miners for including a transaction in a block. Fees are calculated in satoshis per virtual byte (sat/vB). Higher fees prioritize your transaction during mempool congestion. SegWit transactions are cheaper due to the discount on witness data.
FOMO Psychology
Fear Of Missing Out — the emotional urge to buy Bitcoin after a significant price increase. FOMO-driven purchases typically happen near local tops and often result in poor entry points. Awareness of FOMO is essential for disciplined investing.
Fork Core
A change to the Bitcoin protocol that creates a divergence in the blockchain. Soft forks are backward-compatible upgrades (like SegWit or Taproot). Hard forks create an incompatible chain split (like Bitcoin Cash or Bitcoin SV).
Full Node Core
A computer running Bitcoin Core that validates all transactions and blocks against consensus rules. Full nodes enforce the 21 million supply cap, reject invalid blocks, and contribute to network decentralization. Running your own node is a key sovereignty practice.
G
Genesis Block Core
Block 0 of the Bitcoin blockchain, mined by Satoshi Nakamoto on January 3, 2009. The coinbase transaction included the headline "The Times 03/Jan/2009 Chancellor on brink of second bailout for banks" — a timestamp and commentary on the traditional financial system.
H
Halving Core
An event occurring every 210,000 blocks (~4 years) that cuts the block reward in half. Started at 50 BTC/block in 2009, now at 3.125 BTC/block after the April 2024 halving. Halvings continue until the last satoshi is mined around 2140. The mechanism ensures Bitcoin's disinflationary supply schedule.
Hardware Wallet Security
A dedicated physical device designed to securely store Bitcoin private keys offline. Popular models include Ledger, Trezor, Coldcard, and BitBox. Hardware wallets sign transactions without exposing private keys to a potentially compromised computer.
Hashrate Mining
The total computational power secured to the Bitcoin network, measured in exahashes per second (EH/s). A higher hashrate means more security and makes the network more resistant to attacks. Current hashrate exceeds 700 EH/s.
HD Wallet Technical
Hierarchical Deterministic Wallet — a wallet that generates all addresses from a single seed phrase using a mathematical derivation path (BIP32/39/44). This allows infinite addresses from a 12 or 24-word mnemonic, making backups simple and secure.
HODL Culture
A misspelling of "hold" originating from a 2013 Bitcointalk forum post. Now an acronym for "Hold On for Dear Life" — the strategy of holding Bitcoin long-term regardless of market volatility. A core tenet of Bitcoin culture.
I
Inputs Technical
References to previous transaction outputs (UTXOs) being spent in a new transaction. Each input must be signed with the private key corresponding to the address holding the funds. More inputs in a transaction mean higher fees.
K
Keys (Private/Public) Security
The cryptographic foundation of Bitcoin ownership. A private key is a 256-bit number (usually represented as a 64-character hex string or seed phrase) that authorizes spending. The public key is derived from the private key via elliptic curve multiplication. Never share your private key.
L
Lightning Network Layer 2
A second-layer protocol built on top of Bitcoin enabling instant, low-fee payments. Lightning uses payment channels that settle on-chain only when opened or closed. Enables micropayments and everyday Bitcoin transactions without congesting the base layer.
Liquidity Trading
The ability to buy or sell Bitcoin without causing significant price impact. Bitcoin is the most liquid cryptocurrency with deep order books on major exchanges. Liquidity is typically higher during US and European trading hours.
LNURL Lightning
A protocol standard that simplifies Lightning Network interactions by using a single URL to handle payments, withdrawals, and authentication. LNURL-pay allows merchants to receive Lightning payments without generating a new invoice for each transaction.
M
Mempool Core
Short for "memory pool" — the collection of unconfirmed transactions waiting to be included in a block by miners. The mempool is maintained by each node independently. When the mempool is full, fees rise as users compete for block space.
Merkle Tree Technical
A binary tree data structure used in Bitcoin blocks to efficiently verify transactions without downloading the entire block. Each leaf is a transaction hash; parent nodes combine child hashes until a single Merkle Root is produced, stored in the block header.
Miner Mining
A participant running specialized ASIC hardware to compete in the SHA-256 proof-of-work lottery. Miners package transactions into blocks and attempt to find a valid block hash below the current difficulty target. Successful miners receive the block reward and fees.
Mnemonic / Seed Phrase Security
A human-readable representation of a Bitcoin private key, typically 12 or 24 words selected from the BIP39 word list. Seed phrases allow wallets to be backed up and restored. The phrase must be kept secret and stored offline.
Multisig Security
Multi-signature — an address that requires M-of-N signatures to spend funds (e.g., 2-of-3). Multisig wallets distribute trust across multiple devices or people, eliminating single points of failure. Commonly used by businesses, funds, and high-net-worth individuals.
N
Node Core
A computer running the Bitcoin protocol that validates transactions and blocks. Full nodes enforce all consensus rules and relay data to other nodes. Running a full node (Bitcoin Core) is the ultimate way to verify your own transactions without trusting third parties.
Nonce Mining
A 32-bit number included in the block header that miners vary during hashing attempts. Combined with the extraNonce field in the coinbase transaction, it provides the entropy needed to explore the vast search space of possible block hashes.
npub Nostr
A Nostr public key in Bech32 format, used to identify users on the Nostr protocol. The Bitcoin community has heavily adopted Nostr as a censorship-resistant social media layer, with many prominent Bitcoiners using npub handles.
O
Oracle Layer 2
A trusted data feed that provides real-world information (like BTC price or weather data) to blockchain-based smart contracts. Discreet Log Contracts (DLCs) use oracles to enable Bitcoin-based derivatives and prediction markets without a trusted third party.
Output Technical
An element of a Bitcoin transaction that specifies a recipient address and amount of Bitcoin to transfer. Each output becomes a UTXO (Unspent Transaction Output) that can be used as an input in a future transaction. The sum of outputs must be less than or equal to the sum of inputs (difference is the fee).
P
Paper Wallet Security
A physical document containing a Bitcoin address and its private key, typically presented as a QR code. While secure if generated correctly on an offline computer, paper wallets are fragile and outdated — modern hardware wallets are strongly preferred.
Private Key Security
A secret 256-bit number that authorizes Bitcoin spending. Whoever controls the private key controls the Bitcoin. Private keys can be represented as hex, WIF (Wallet Import Format), or a seed phrase. Compromised private key = lost Bitcoin.
Proof of Work Core
The consensus mechanism securing Bitcoin. Miners compete to find a block hash below a target threshold by varying the nonce in the block header. The first miner to find a valid hash broadcasts their block to the network. PoW makes rewriting history computationally prohibitive.
Public Key Technical
Derived from the private key using elliptic curve multiplication (secp256k1), the public key is used to generate Bitcoin addresses. Unlike addresses (which are hashed), public keys are not directly exposed until funds are spent from an address.
R
RBF Technical
Replace-By-Fee — a feature allowing a sender to replace an unconfirmed transaction with a higher-fee version. RBF is useful when the mempool is congested and a transaction is stuck with low fees. Most modern wallets support RBF by default.
Runes Protocol
A fungible token protocol built on Bitcoin introduced by Casey Rodarmor (creator of Ordinals). Runes use the UTXO model for efficient token transfers without creating excessive "junk" UTXOs, unlike BRC-20 tokens.
S
Satoshi (SAT) Core
The smallest unit of Bitcoin, equal to 0.00000001 BTC or 1/100,000,000 of a full Bitcoin. Named after Bitcoin's pseudonymous creator. As Bitcoin price rises, transactions are increasingly denominated in satoshis rather than BTC.
SegWit Technical
Segregated Witness — a soft fork activated in August 2017 that separated (segregated) witness data (signatures) from the transaction data. This effectively increased block capacity, fixed transaction malleability, and enabled second-layer protocols like Lightning Network.
Self-Custody Security
The practice of holding your own Bitcoin private keys rather than trusting a third party (exchange, custodian). The mantra "not your keys, not your coins" emphasizes that only self-custody gives you true ownership of Bitcoin.
Signature Technical
A cryptographic proof created by signing a transaction with a private key. ECDSA (Elliptic Curve Digital Signature Algorithm) signatures prove ownership without revealing the private key. Schnorr signatures (enabled by Taproot) allow multi-signature aggregation for greater efficiency.
SOIL Culture
Standard Of Independent Living — a Bitcoin-centric alternative to COLA (Cost Of Living Adjustments). Refers to the practice of measuring your standard of living in Bitcoin terms rather than depreciating fiat currency.
Soft Fork Core
A backward-compatible upgrade to the Bitcoin protocol where new rules are stricter than old rules. Old nodes can still validate new blocks (but may not enforce the new rules). Examples: SegWit, Taproot, OP_CHECKSEQUENCEVERIFY.
T
Taproot Technical
A Bitcoin soft fork activated in November 2021 (block 709,632) that introduced Schnorr signatures and MAST (Merkleized Abstract Syntax Trees). Taproot improves privacy by making complex smart contracts indistinguishable from simple payments on-chain, and reduces transaction sizes.
Transaction Core
A signed data structure that transfers Bitcoin from one address to another. Transactions consume UTXOs as inputs and create new UTXOs as outputs. Each transaction is broadcast to the network, validated by nodes, and eventually included in a block by miners.
U
UTXO Core
Unspent Transaction Output — a discrete piece of Bitcoin that you own. Your Bitcoin balance is the sum of all UTXOs controlled by your keys. Each UTXO must be spent in its entirety (the "change" goes to a new change address). Managing UTXOs is called coin control.
V
Vanity Address Culture
A Bitcoin address that contains a specific pattern or word (e.g., bc1qdeadbeef...). Generating vanity addresses requires brute-forcing millions of keys until one produces the desired pattern. Vanity addresses are a flex but offer no additional security.
W
Wallet Core
Software or hardware that manages Bitcoin private keys and facilitates transactions. Wallets don't "store" Bitcoin — they store keys that control UTXOs on the blockchain. Types include: hardware wallets (Ledger/Trezor), software wallets (Sparrow/Electrum), and mobile wallets (BlueWallet/Muun).
Whale Trading
An entity holding a large amount of Bitcoin (typically 1,000+ BTC). Whale movements can impact market prices. Whale watching — tracking large transactions on-chain — is a popular analysis technique for anticipating market moves.
X
xpub Technical
Extended Public Key — a master public key from which all descendant public keys in an HD wallet can be derived. Sharing an xpub allows others to view your wallet's transaction history (watch-only) without being able to spend funds. Watch-only wallets use xpubs.
Z
Zero-Conf Core
Zero-Confirmation — accepting a Bitcoin transaction before it has been included in a block. Zero-conf transactions carry risk of double-spends, especially for high-value payments. Lightning Network solves this problem with instant, irreversible payments.
