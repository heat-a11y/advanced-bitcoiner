🧬 How Bitcoin Works (Deep)

🔗

UTXOs

Bitcoin doesn't use "balances." You own UTXOs — like digital coins. A transaction spends old UTXOs and creates new ones.

⛓️

The Mempool

Unconfirmed transactions wait in the mempool. Miners pick the most profitable ones. Higher fee = faster confirmation.

⚡

Lightning Network

A layer-2 that enables instant, near-zero fee payments. Open a channel, pay off-chain, settle on-chain when done.

🔑

HD Wallets

A single seed phrase (12/24 words) derives every address. BIP39 mnemonic + BIP32 derivation path = infinite keys.

🔄

Difficulty Adjustment

Every 2016 blocks (~2 weeks), Bitcoin adjusts mining difficulty to keep block time at ~10 minutes.

🕵️

Privacy

Bitcoin is pseudonymous, not anonymous. Each address is public. Best practices: avoid address reuse, use CoinJoin.

📜

Script & Smart Contracts

Bitcoin has a scripting language (Script). Time-locks, multisig, and Taproot enable advanced spending conditions.

🎨

Ordinals

Ordinals inscribe data onto individual satoshis. Enabled by Taproot — turns every sat into a potential NFT.

🌐 Running a Node — Be the Network

Running your own Bitcoin node gives you sovereignty. You verify transactions yourself instead of trusting someone else.

  1. Choose your node software

    Bitcoin Core is the reference implementation. Download from bitcoin.org.

  2. Hardware matters

    You need ~600GB+ free disk space (pruning reduces it to ~10GB). 8GB RAM recommended. A Raspberry Pi 4 can run a pruned node.

  3. Initial Block Download (IBD)

    Syncing from scratch takes 1-3 days depending on hardware. Be patient — every block is verified from genesis.

  4. Connect your wallet

    Use your own node with Electrum, Sparrow, or Specter. No more leaking data to third-party servers.

  5. Open ports (optional)

    Port 8333 lets other nodes connect to you. You become part of the network backbone!

🔬 Advanced Myths

❌ "Bitcoin can be copied / a better coin will replace it"
Bitcoin has the strongest network effect, the most hashrate, the most developers, and the most liquidity. A clone has none of this. Network effects are incredibly hard to replicate.
❌ "Bitcoin isn't private enough"
True that base-layer Bitcoin is pseudonymous, but tools like CoinJoin, Lightning, and payjoin significantly improve privacy. Bitcoin's transparency is a feature — you can verify the supply.
❌ "The 21M cap can be changed"
Technically possible, but it would require near-universal consensus. Nodes enforce the supply rule. Any chain changing the 21M cap would be rejected by the community as a different asset.
❌ "Lightning Network is centralized"
Anyone can run a Lightning node. While routing tends toward larger nodes, new protocols (like trampoline routing) and improved pathfinding are making it more decentralized over time.
❌ "Quantum computers will break Bitcoin"
Elliptic curve cryptography (ECDSA) is vulnerable to Shor's algorithm. But Bitcoin can upgrade to quantum-resistant signatures (like Lamport or Winternitz) when needed — it's a software upgrade.

📖 Glossary

UTXO
Unspent Transaction Output. Like a coin in your pocket. Your wallet balance is the sum of all UTXOs you can spend.
Mempool
The waiting room for unconfirmed transactions. Miners select from here. Higher fee = bumped to the front.
BIP39 / BIP32
BIP39 defines the 12/24-word seed phrase. BIP32 defines hierarchical deterministic (HD) key derivation — one seed → infinite addresses.
Derivation Path
A path like m/84'/0'/0'/0/0 that tells your wallet which key to use. Different paths = different accounts.
SIGHASH
Signature Hash — defines which parts of a transaction are signed. Enables complex smart contracts like CoinJoin.
Taproot
A 2021 upgrade (BIP341) that makes complex scripts look like simple payments. Improves privacy and reduces fees for smart contracts.
CoinJoin
A privacy technique where multiple users combine their transactions. An observer can't tell who paid whom.
Payjoin (BIP78)
A privacy-improving technique where the recipient adds a payment of their own in the same transaction, breaking common chain-analysis heuristics.
